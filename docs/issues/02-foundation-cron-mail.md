## 📝 タイトルで伝えたいこと（一言で）

タスク進捗の定期メール送信を自動で動くようにする（cron 配線）

---

## 📖 くわしい内容

日次 / 週次のメール送信は、設定 UI・スケジュール判定（`isDue`）・重複防止（`last_sent_at`）
まで実装済みですが、`POST /api/send-summary`（`{ scheduled: true }`）を定期的に呼び出す
仕組みがないため、**実際には自動送信されません**。

- `vercel.json` に cron 設定がない（現状 `{"regions":["hnd1"]}` のみ）
- `supabase/functions/send-summary/` は古い重複実装が残っており、本番の Next.js API とは
  内容が食い違っている（旧 `recipients` 列しか見ない等）

設定した頻度どおりにメールが自動送信される状態にします。

## 📌 補足・メモ

- cron 実行の粒度は送信時刻（`send_time`）に合わせる（例: 毎時実行し `isDue` で対象を絞る）。
- **担当境界に注意**: メール本文・タスクデータは別担当（タスク / メール）の領域と重なるため、
  cron の配線範囲について事前にすり合わせる。
